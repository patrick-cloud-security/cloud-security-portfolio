# Cloud Threat / Control Matrix

This matrix summarises common AWS cloud attack paths, the permissions or misconfigurations that enable them, and defensive controls that reduce blast radius. It is part of my cloud security portfolio and is based on hands-on AWS IAM, EC2 workload identity, S3, IMDSv2, and privilege escalation labs.

| Cloud Area | Threat / Attack Path | Dangerous Permission or Misconfiguration | Impact | Preventive Control | Detection / Monitoring | Related Lab |
|---|---|---|---|---|---|---|
| EC2 Workload Identity | Stolen EC2 role credentials replayed outside the instance | EC2 instance role credentials exposed through IMDS after workload compromise | Attacker can call AWS APIs as the EC2 role until credentials expire | Require IMDSv2, attach least-privilege roles, avoid sensitive permissions on workload roles | CloudTrail events from assumed role, unusual source IP, denied API calls from workload role | EC2 Workload Identity / IMDSv2 Lab |
| IAM | Privilege escalation through role passing | `iam:PassRole` combined with `ec2:RunInstances` | Attacker can launch compute with a more privileged role and retrieve its credentials | Scope `iam:PassRole` to approved role ARNs, restrict `ec2:RunInstances`, use permission boundaries/SCPs | CloudTrail `PassRole` and `RunInstances` events, new instance profile usage | IAM Attack Path Lab |
| S3 | Bucket discovery and data exfiltration | `s3:ListAllMyBuckets`, `s3:ListBucket`, `s3:GetObject` granted too broadly | Attacker can discover buckets, list objects, and read sensitive data | Scope bucket/object access by ARN and prefix, separate read/write roles, block public access | S3 data events, unusual object reads, access from unexpected role/session | IAM/S3 Access Lab |
| Secrets Manager | Secret theft after workload compromise | `secretsmanager:GetSecretValue` on `Resource="*"` | Attacker can retrieve database passwords, API keys, tokens, or other stored secrets | Allow access only to specific secret ARNs required by the workload | CloudTrail `GetSecretValue`, secret access from unexpected role or source IP | EC2 Workload Identity / IMDSv2 Lab |
| SSM | Remote command execution on managed instances | `ssm:SendCommand` granted to broad targets | Attacker can execute commands on EC2 instances without SSH | Restrict `ssm:SendCommand` to admin roles and tagged instances, require approval workflows | CloudTrail `SendCommand`, SSM command history, unexpected command execution | IAM Attack Path Lab |
| Terraform | Credential exposure through state files | Terraform state committed to Git or stored insecurely | Exposed credentials, resource IDs, ARNs, sensitive outputs, infrastructure mapping | Never commit state, use remote encrypted backend, restrict backend access, avoid sensitive outputs | Secret scanning, GitHub push protection, backend access logs | Terraform State Exposure Lab |
| IAM Reconnaissance | Account role enumeration | `iam:ListRoles`, `iam:GetRole`, `iam:ListPolicies` | Attacker can map privilege boundaries and identify escalation targets | Avoid granting identity enumeration to workload roles unless required | CloudTrail IAM list/get events from unusual roles | IAM Attack Path Lab |
| EC2 Reconnaissance | Infrastructure discovery | `ec2:DescribeInstances`, `ec2:DescribeSecurityGroups`, `ec2:DescribeSubnets` | Attacker can map network layout, exposed services, instance profiles, and targets | Avoid broad describe permissions on workload roles unless needed | CloudTrail EC2 Describe events, unusual enumeration patterns | EC2 Workload Identity / IMDSv2 Lab |
| CloudWatch Logs | Sensitive information disclosure through logs | Broad `logs:GetLogEvents` or secrets written to logs | Attacker may recover tokens, errors, internal endpoints, or credentials from logs | Avoid logging secrets, scope log access, use redaction and log retention policies | CloudTrail log access, unusual log group reads | Future Lab |
| KMS | Decryption of protected data | Broad `kms:Decrypt` access | Attacker can decrypt data protected with KMS if they can access ciphertext | Scope key usage by principal, encryption context, and specific service paths | CloudTrail `Decrypt`, KMS key usage anomalies | Future Lab |
